# Роли и права доступа

## 1. Иерархия ролей

```
Администратор (системный)
    │
    └── Владелец (бизнес-администратор)
            │
            ├── Эксперт
            │
            └── Пользователь
```

## 2. Детальное описание ролей

### 2.1 Администратор (Administrator)

**Назначение:** Техническое управление системой.

**Права:**
- Управление системными настройками
  - Настройка базы данных
  - Настройка email-сервера
  - Управление CORS, CSP
  - Настройка платёжных систем
- Просмотр системных логов
- Управление миграциями БД
- Техническая диагностика

**Ограничения:**
- Не имеет доступа к бизнес-данным
- Не может модерировать контент
- Не может управлять пользователями напрямую

**UI-интерфейс:** Системная панель (`/admin`)

---

### 2.2 Владелец (Owner)

**Назначение:** Полное управление бизнесом платформы.

**Права:**
- Управление пользователями
  - Просмотр всех пользователей
  - Блокировка/разблокировка
  - Изменение ролей
  - Назначение модераторов
- Модерация контента
  - Одобрение/отклонение экспертов
  - Управление доступностью слотов
- Финансовый контроль
  - Просмотр всех платежей
  - Управление комиссиями
  - Выплаты экспертам
- Аналитика
  - Статистика по бронированиям
  - Финансовые отчёты
  - Отчёты по активности

**UI-интерфейс:** Dashboard (`/dashboard`, `/accounts`)

---

### 2.3 Эксперт (Expert)

**Назначение:** Экспертная деятельность на платформе.

**Права:**
- Управление профилем эксперта
  - Фото, описание, специализация
  - Контактная информация
  - Часовой пояс
- Управление расписанием
  - Публикация слотов свободного времени
  - Установка стоимости
  - Отмена слотов
- Взаимодействие с пользователями
  - Просмотр записавшихся
  - Комментарии
- Финансы
  - Просмотр своих доходов
  - История выплат

**UI-интерфейс:** Teaching panel (`/expert`)

---

### 2.4 Пользователь (User)

**Назначение:** Использование платформы для бронирования консультаций.

**Права:**
- Управление профилем
  - Личные данные
  - Часовой пояс
  - Уведомления
- Поиск и выбор
  - Поиск экспертов
  - Фильтры по стоимости, времени
- Бронирование и оплата
  - Бронирование слотов
  - Оплата онлайн
- История
  - Расписание своих консультаций
  - История бронирований
- Обратная связь
  - Комментарии
  - Рейтинг экспертов

**UI-интерфейс:** User panel (`/profile`, `/bookings`)

## 4. Смена ролей

### Назначение эксперта

1. Пользователь получает тип аккаунта `expert`
2. Создаёт профиль эксперта
3. Владелец/модератор рассматривает профиль
4. При одобрении — флаг `IS_APPROVED = 1`
5. Эксперт получает доступ к `/expert`

### Назначение модератора

1. Только Владелец или Админ может назначить модератора
2. Устанавливается флаг `IS_MODERATOR = 1`

### Назначение владельца

1. Только Администратор может назначить Владельца
2. Назначение через системный интерфейс или напрямую в БД

## 5. Хранение ролей в системе

```php
// Роли в базе данных (таблица db_accounts)
type ENUM('user', 'expert')

// Флаги в db_account_data
IS_ADMIN, IS_OWNER, IS_MODERATOR, IS_APPROVED, IS_DISABLED

// Проверка в middleware
if ($account->isAdmin()) { /* системный доступ */ }
if ($account->isOwner()) { /* бизнес-администрирование */ }
if ($account->isExpert()) { /* экспертная панель */ }
// По умолчанию — user
```
