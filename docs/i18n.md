# i18n: единый источник для backend и frontend

## Как это устроено

Переводы хранятся в PHP-файлах `*I18nDataRu.php` и `*I18nDataEn.php`. Backend использует классы Garnet i18n, а `php garnet prepare` генерирует из тех же ключей TypeScript-файлы в `Front/I18nGen/`.

Для IRabi это означает:

- PHP/Twig вызывают typed PHP i18n API;
- React/TypeScript вызывают typed `I18nForeground`/`I18nFramework` API;
- ключи должны совпадать между RU и EN;
- generated-файлы нельзя редактировать вручную;
- после изменения PHP-каталогов нужно выполнить `php garnet prepare`, затем `php garnet build`.

Это единый источник на уровне каждого bundle, а не один гигантский словарь на всё приложение. Такой вариант сохраняет изоляцию framework, app и feature-модулей.

## Проверка текущего состояния

Проверка снимка от 2026-07-15 показала:

- `ForegroundI18nDataRu.php`: 898 ключей;
- `ForegroundI18nDataEn.php`: 887 ключей;
- `I18nForeground.ts`: 898 typed методов;
- в английском каталоге отсутствуют 11 ключей:
  `Admin_Flag_Approve`, `Admin_Flag_Disable`, `Admin_Flag_Enable`,
  `Admin_Flag_GrantAdmin`, `Admin_Flag_GrantModerator`,
  `Admin_Flag_GrantOwner`, `Admin_Flag_RevokeAdmin`,
  `Admin_Flag_RevokeApproval`, `Admin_Flag_RevokeModerator`,
  `Admin_Flag_RevokeOwner`, `Slot_Save`.

Это блокирует заявление о полной двуязычной поставке. Нужно добавить английские значения, выполнить `php garnet prepare`, `php garnet build`, `php garnet build:check` и проверить UI на обоих языках.

## Правило изменения переводов

1. Изменить пару PHP-каталогов RU/EN.
2. Проверить одинаковый набор ключей.
3. Выполнить `php garnet prepare`.
4. Не редактировать `Front/I18nGen/*` вручную.
5. Запустить TypeScript check и production build.
6. Добавить smoke/E2E-проверку новой пользовательской строки, если она относится к критическому сценарию.

## Рекомендуемый CI-gate

Сделать parity-check, который завершает CI с ошибкой при:

- ключе только в одном языке;
- ключе PHP, отсутствующем в generated TypeScript;
- различающихся `%s` placeholder-ах;
- вызове неизвестного typed i18n метода.

Подробное описание механизма Garnet: [`garnet-framework/docs/i18n.md`](../../../garnet-framework/docs/i18n.md).
