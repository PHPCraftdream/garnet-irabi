# Expert API

## Контроллер

**Класс:** `TeachingController`
**URL:** `/expert/`
**Доступ:** только для экспертов (middleware `expertOnly`)

## Эндпоинты

### GET /expert/ — Главная страница эксперта

Отображает страницу управления слотами и профилем.

### GET /expert/~slots — Список слотов

Отображает все слоты эксперта с возможностью создания одиночных и пакетных слотов.

### POST /expert/~slots — Создание одиночного слота

**Параметры (FormData):**

| Параметр | Тип | По умолч. | Описание |
|----------|-----|-----------|----------|
| `date` | string | — | Дата (YYYY-MM-DD), обязательно |
| `time` | string | — | Время (HH:MM), обязательно |
| `duration` | int | 60 | Длительность в минутах |
| `cost` | int | 0 | Стоимость |
| `is_online` | int | 1 | Онлайн (1) или офлайн (0) |
| `location` | string | '' | Место проведения |

**Ответ:** `{ success: true, slot_id: N }`

### POST /expert/~batchPreview — Предпросмотр пакетного создания

Анализирует диапазон дат с учётом еврейского календаря (Шаббат, праздники) и возвращает доступные/запрещённые даты + предложенное распределение слотов.

**Параметры (FormData):**

| Параметр | Тип | По умолч. | Описание |
|----------|-----|-----------|----------|
| `start_date` | string | — | Начальная дата, обязательно |
| `end_date` | string | — | Конечная дата, обязательно |
| `count` | int | 4 | Желаемое количество слотов |
| `duration` | int | 60 | Длительность в минутах |

**Ответ:**
```json
{
    "availableDates": [{"date": "2026-04-05", "hebrew": "7 Nisan 5786"}],
    "restrictedDates": [{"date": "2026-04-04", "reason": "Шаббат", "hebrew": "..."}],
    "proposedDates": ["2026-04-05", "2026-04-07"],
    "existingSlots": [{"date": "2026-04-05", "time": "10:00", "duration_min": 60}],
    "totalAvailable": 20,
    "totalRestricted": 10
}
```

### POST /expert/~batchSlots — Пакетное создание слотов

Создаёт несколько слотов за один запрос. Проверяет пересечения с существующими слотами на сервере.

**Параметры (FormData):**

| Параметр | Тип | По умолч. | Описание |
|----------|-----|-----------|----------|
| `slots` | JSON string | — | Массив слотов `[{date, time, duration}]` |
| `cost` | int | 500 | Стоимость каждого слота |

**Ответ:**
```json
{
    "success": true,
    "created": 3,
    "overlaps": [{"date": "2026-04-05", "time": "10:00", "reason": "Overlaps with existing slot"}]
}
```

## Проверка пересечений (Overlap Detection)

### Клиентская сторона (предупреждение)

В Twig-шаблоне функция `hasOverlap()` проверяет пересечение предлагаемого слота с существующими:

```
overlap = proposed_start < existing_end && proposed_end > existing_start
```

Показывает предупреждение, но не блокирует отправку.

### Серверная сторона (авторитетная)

В `post__batchSlots` — запрос существующих слотов эксперта, проверка пересечений. Пересекающиеся слоты пропускаются и возвращаются в массиве `overlaps`.

## Еврейский календарь

Класс `SlotDateFilter` фильтрует даты по еврейскому календарю:
- Шаббат (суббота)
- Йом Тов (праздники)
- Посты
- Рош Ходеш

Используется для ограничения выбора дат при пакетном создании слотов.
