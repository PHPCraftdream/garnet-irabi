# Workflow процессов

## 1. Процессы эксперта

### 1.1 Публикация слотов времени

```
┌─────────────────────────────────────────────────────────────────┐
│                  ПУБЛИКАЦИЯ СЛОТОВ ВРЕМЕНИ                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Вариант A: Единичный слот                                      │
│  ─────────────────────────                                      │
│  1. Выбрать дату и время                                        │
│  2. Указать длительность                                        │
│  3. Установить стоимость                                        │
│  4. Определить формат (онлайн/офлайн)                           │
│  5. Опубликовать                                                │
│                                                                 │
│  Вариант B: Массовое создание (batch)                           │
│  ─────────────────────────────                                  │
│  1. Выбрать период (дата начала - дата окончания)               │
│  2. Указать количество слотов                                   │
│  3. Указать длительность                                        │
│  4. Установить стоимость                                        │
│  5. Система фильтрует даты по еврейскому календарю              │
│  6. Предпросмотр предложенных дат                               │
│  7. Подтверждение → создание слотов                             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 Управление профилем эксперта

```
┌─────────────────────────────────────────────────────────────────┐
│                 УПРАВЛЕНИЕ ПРОФИЛЕМ ЭКСПЕРТА                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. Заполнить основную информацию:                              │
│     • Отображаемое имя                                          │
│     • Описание (bio)                                            │
│     • Специализация                                             │
│     • Фото                                                      │
│     ↓                                                           │
│  2. Профиль создан со статусом is_approved = 0                  │
│     ↓                                                           │
│  3. Модератор/владелец рассматривает заявку                     │
│     ↓                                                           │
│  4. При одобрении (IS_APPROVED = 1):                            │
│     → Слоты отображаются пользователям                          │
│     → Профиль виден в каталоге                                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 2. Процессы пользователя

### 2.1 Бронирование слота времени

```
┌─────────────────────────────────────────────────────────────────┐
│                  БРОНИРОВАНИЕ СЛОТА ВРЕМЕНИ                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. Пользователь ищет эксперта или свободные слоты              │
│     ↓                                                           │
│  2. Выбирает подходящий слот:                                   │
│     • Дата и время                                              │
│     • Стоимость                                                 │
│     • Формат (онлайн/офлайн)                                    │
│     ↓                                                           │
│  3. Нажимает "Забронировать"                                    │
│     ↓                                                           │
│  4. Проверяется баланс (balance >= cost)                        │
│     ↓                                                           │
│  5. Списывается стоимость, создаётся бронирование               │
│     ↓                                                           │
│  6. Слот получает статус "booked" (если последнее место)        │
│     ↓                                                           │
│  7. Эксперт получает уведомление о бронировании                 │
│     ↓                                                           │
│  8. Пользователь получает подтверждение и напоминание           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 Отмена бронирования

```
┌─────────────────────────────────────────────────────────────────┐
│                  ОТМЕНА БРОНИРОВАНИЯ                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. Пользователь или эксперт инициирует отмену                  │
│     ↓                                                           │
│  2. Бронирование переходит в статус "cancelled"                 │
│     ↓                                                           │
│  3. Средства возвращаются пользователю (booking_refund)         │
│     ↓                                                           │
│  4. Средства списываются у эксперта (booking_refund)            │
│     ↓                                                           │
│  5. Слот освобождается (booked → free, если место освободилось) │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 3. Процессы владельца

### 3.1 Модерация экспертов

```
┌─────────────────────────────────────────────────────────────────┐
│                     МОДЕРАЦИЯ ЭКСПЕРТОВ                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. Владелец видит список не одобренных экспертов               │
│     ↓                                                           │
│  2. Открывает профиль эксперта                                  │
│     ↓                                                           │
│  3. Проверяет:                                                  │
│     • Соответствие правилам платформы                           │
│     • Полноту информации                                        │
│     • Качество профиля                                          │
│     ↓                                                           │
│  4. Принимает решение:                                          │
│     ┌─────────────┬─────────────┐                                │
│     │  Одобрить   │  Отклонить  │                                │
│     └─────────────┴─────────────┘                                │
│           │              │                                       │
│           ▼              ▼                                       │
│     IS_APPROVED=1   IS_APPROVED остаётся 0                      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 Управление пользователями

```
┌─────────────────────────────────────────────────────────────────┐
│                 УПРАВЛЕНИЕ ПОЛЬЗОВАТЕЛЯМИ                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Действия с пользователями:                                     │
│                                                                 │
│  • Просмотр списка всех пользователей                           │
│  • Фильтрация по роли, статусу, дате регистрации               │
│  • Поиск по имени, email                                        │
│                                                                 │
│  Действия над пользователем:                                    │
│                                                                 │
│  ┌─────────────────┐                                            │
│  │ Заблокировать   │ → IS_DISABLED = 1                         │
│  └─────────────────┘                                            │
│  ┌─────────────────┐                                            │
│  │ Разблокировать  │ → IS_DISABLED = 0                         │
│  └─────────────────┘                                            │
│  ┌─────────────────┐                                            │
│  │ Назначить       │ → type = 'expert'                         │
│  │ экспертом       │   + создать expert_profile                │
│  └─────────────────┘                                            │
│  ┌─────────────────┐                                            │
│  │ Одобрить        │ → IS_APPROVED = 1                         │
│  │ эксперта        │                                            │
│  └─────────────────┘                                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 4. Финансовые процессы

### 4.1 Схема оплаты

```
┌─────────────────────────────────────────────────────────────────┐
│                     СХЕМА ОПЛАТЫ                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Пользователь платит X                                          │
│       │                                                         │
│       ▼                                                         │
│  Баланс пользователя уменьшается на X (booking_invoice)         │
│       │                                                         │
│       ▼                                                         │
│  Баланс эксперта увеличивается на X (booking_payment)           │
│       │                                                         │
│       ▼                                                         │
│  Записи в BalanceLedger (иммутабельный журнал)                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4.2 Возврат средств

```
┌─────────────────────────────────────────────────────────────────┐
│                    ВОЗВРАТ СРЕДСТВ                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  При отмене бронирования:                                       │
│  1. BookingSM → cancelled                                       │
│  2. LedgerSM: +booking_refund (пользователь +cost)              │
│  3. LedgerSM: +booking_refund (эксперт −cost)                  │
│  4. BalanceSM: пересчёт обоих аккаунтов                         │
│  5. TimeSlotSM: booked → free (если место освободилось)         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 5. Уведомления

### 5.1 Типы уведомлений

| Событие | Пользователь | Эксперт | Владелец |
|---------|:------------:|:-------:|:--------:|
| Бронирование слота | ✅ | ✅ | ❌ |
| Отмена бронирования | ✅ | ✅ | ❌ |
| Напоминание о консультации | ✅ | ✅ | ❌ |
| Одобрение/отклонение эксперта | ❌ | ✅ | ❌ |
| Оплата | ✅ | ✅ | ❌ |
| Возврат | ✅ | ✅ | ❌ |

### 5.2 Каналы уведомлений

- Email (обязательно)
- Push-уведомления (опционально)
