# IRabi — Система стейт-машин

## Принцип

Приложение IRabi — это **система взаимодействующих стейт-машин**. Каждая сущность (пользователь, бронирование, слот, баланс) — отдельная стейт-машина со своими состояниями и правилами перехода. Роли не «права доступа» — это **акторы**, которые могут инициировать переходы в соседних или подчинённых машинах.

## Иерархия машин

```
┌─────────────────────────────────────────────────────────┐
│  AccountSM (Admin)        ← управляет всем              │
│  ┌───────────────────────────────────────────────────┐  │
│  │  PlatformSM (Owner)    ← управляет ролями         │  │
│  │  ┌─────────────────────────────────────────────┐  │  │
│  │  │  ModerationSM (Moderator) ← одобрения       │  │  │
│  │  │  ┌───────────────────────────────────────┐  │  │  │
│  │  │  │  ExpertSM   ← слоты и профиль         │  │  │  │
│  │  │  │  ┌─────────────────────────────────┐  │  │  │  │
│  │  │  │  │  UserSM     ← бронирования       │  │  │  │  │
│  │  │  │  └─────────────────────────────────┘  │  │  │  │
│  │  │  └───────────────────────────────────────┘  │  │  │
│  │  └─────────────────────────────────────────────┘  │  │
│  └───────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘

Соседние машины (реагируют на события сверху):
  TimeSlotSM ←→ BookingSM ←→ BalanceSM ←→ LedgerSM
  ExpertProfileSM
```

## Стейт-машины системы

| Машина | Таблица | Состояния | Документ |
|--------|---------|-----------|----------|
| **AccountSM** | `db_accounts` + `db_account_data` | `unregistered → registered → approved → disabled` | [admin.md](admin.md) |
| **ExpertProfileSM** | `ir_expert_profiles` | `not_approved ↔ approved` | [expert.md](expert.md) |
| **TimeSlotSM** | `ir_time_slots` | `free → booked → completed / cancelled` | [expert.md](expert.md) |
| **BookingSM** | `ir_bookings` | `pending → confirmed → completed / cancelled` | [user.md](user.md) |
| **BalanceSM** | `ir_account_balance` | `balance: INT (изменяется через LedgerSM)` | [user.md](user.md) |
| **LedgerSM** | `ir_balance_ledger` | иммутабельный журнал записей | [user.md](user.md) |
| **AdminActionLogSM** | `ir_admin_action_log` | иммутабельный аудит-лог | [admin.md](admin.md) |

## Правила взаимодействия

1. **Верхняя машина инициирует переход в нижней** — Admin меняет флаг `IS_APPROVED` → каскадно меняет `ExpertProfileSM`.
2. **Соседние машины реагируют синхронно** — `BookingSM` переходит в `pending` → одновременно `BalanceSM` дебетует пользователя и кредитует эксперта через `LedgerSM`.
3. **Нижняя машина ограничивает верхнюю** — `TimeSlotSM` в состоянии `booked` запрещает создание новых `BookingSM` для этого слота (если заняты все места).
4. **Иммутабельные машины** — `LedgerSM` и `AdminActionLogSM` только накапливают записи, никогда не изменяют и не удаляют их.
